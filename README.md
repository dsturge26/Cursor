# Enterprise AI Deployment Readiness Diagnostic

An independent, static GitHub Pages project built by Drew Sturgell. The app is a single-page interactive diagnostic for enterprise AI coding-tool deployment readiness.

It helps a user assess:

- Engineering team size
- Current AI coding-tool maturity
- Rollout stage
- Primary blocker
- Executive sponsor strength
- Developer enablement maturity
- SDLC integration maturity
- Change management readiness
- Measurement discipline

After submission, the diagnostic generates an overall readiness score, deployment maturity level, recommended rollout motion, top adoption risks, risk mitigation plan, success metrics, enablement plan, executive narrative, expansion opportunities, and a 30/60/90-day action plan.

## Run Locally

No build step is required.

Open `index.html` directly in a browser, or serve the folder with any simple local web server.

## Publish With GitHub Pages

1. Create a new GitHub repository.
2. Push this project to the repository.
3. In GitHub, open the repository settings.
4. Go to **Pages**.
5. Under **Build and deployment**, choose **Deploy from a branch**.
6. Select the `main` branch and the `/root` folder.
7. Save the setting and wait for GitHub Pages to publish the site.

Because `index.html` is at the repository root and the project has no backend or build step, it is suitable for GitHub Pages.
